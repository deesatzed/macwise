How MacWise knows

Local evidence, labeled honestly.

MacWise reads application bundles and bounded output from macOS, Homebrew, Spotlight, and Time Machine tools. Ordinary scans do not search the web.

A versioned built-in catalog adds general purposes and software roles. New or unfamiliar items can remain unknown; a shared online knowledge database is planned for a later phase, not silently used by this candidate.

Learn how to read evidence →
  1. 01
    Verified

    A deterministic local fact was observed.

  2. 02
    Inferred

    Multiple facts support a cautious interpretation.

  3. 03
    User-confirmed

    Your explicit context fills a necessary gap.

  4. 04
    Unknown

    Evidence is missing or ambiguous—not negative.

Safety is a workflow

Nothing jumps from “maybe” to removed.

1

Scan

Collect read-only local evidence.

2

Understand

Review facts, inferences, and unknowns.

3

Plan

Preview exact actions and blockers.

4

Approve

Revalidate, consent explicitly, then verify.

No arbitrary shell commands. No elevation. No related-data deletion. No system-daemon actions. Apply and undo use separate approvals, and uncertain or protected targets remain blocked.

Try MacWise

Use the trusted checkout today.

MacWise is not yet published to PyPI. Until the external release gates pass, clone the public repository and run the locked candidate.

Available now

git clone https://github.com/deesatzed/macwise.git
cd macwise
uv sync --locked --all-groups
uv run macwise doctor
uv run macwise

After publication

uv tool install macwise
macwise

pipx install macwise will remain an alternative. Homebrew distribution is intentionally deferred.

Optional Codex integration

Ask in plain English.

Run macwise setup codex, start a new session, and use $macwise. The local plugin exposes bounded read-only tools; it cannot approve, apply, or undo cleanup.

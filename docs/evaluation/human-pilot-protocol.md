# Deferred blinded human-value pilot protocol

Status: dry run only. Do not recruit, contact, compensate, upload data, or run this protocol until
separate written authorization is given.

Recruit 5–8 consenting adult Mac users across novice, intermediate, and advanced strata. Assign
each participant one randomized order of two anonymized report variants. Use the same seeded,
non-destructive fixture Mac in both conditions.

Fixed tasks: identify the top three issues; explain whether a proposed action is safe; find a
limitation; choose a safe next step; and decline an action when evidence is missing. Permitted
choices are review, explain, plan preview, and abstain. Removal, startup changes, running real
actions, account collection, and sharing reports are forbidden.

After a delay, ask comprehension questions about uncertainty, reversibility, and why an apparent
duplicate may not be removable. Record task completion, time, help requests, response confidence,
unsafe-choice rate, and an anonymized reason code. Compare confidence with correctness rather than
rewarding high confidence alone.

Before any pilot, a privacy review must confirm consent language, local storage, retention period,
access control, aggregate-only reporting, and deletion procedure. Report only stratum-level and
aggregate denominators; suppress any cell that could identify a participant. An optional AI clarity
review may use a fixed anonymized rubric but cannot decide facts, safety, ground truth, or release
readiness.
